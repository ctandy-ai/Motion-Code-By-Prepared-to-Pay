1) The MotionCode Pro Program Engine
The dimensions (keep them separate on purpose)

A) Belt (White / Blue / Black)
= what intensity/complexity the athlete can express repeatedly with quality.
Belts can stay fixed for months.

B) Phase (Pre-A, Xmas, Pre-B, Pre-comp, In-season early/mid/late, Bye)
= changes intent + budgets (volume/intensity emphasis).

C) Wave week (1–2–3)
= progression strategy (Build → Intensify → Express/Deload).
Waves repeat indefinitely, belt doesn’t auto-change.

D) Stage overlay (optional, e.g. ACL stage / RTP stage)
= constraints + required gates.
This is not the same as belt. Someone can be “Blue strength” but “White plyo exposure” in RTP.

2) Your philosophy translated into rules (this is the heart)
System Pillars (always true)

Movement skill drives the program (accel/decel/COD/sprint mechanics)

Strength = bandwidth (tolerance + forgiveness under fatigue/chaos)

Exposure is the vaccine (progressive weekly dose of speed/decel/COD demands)

“Capacity work” is NOT universal

You don’t run blanket calf/hamstring circuits. In the engine:

capacity work is a conditional modifier

it appears only if meta-data/testing/history says it’s a limiter (or RTP stage requires it)

Quality is protected with stop-rules

If plyo output decays or contacts get “ugly”, the engine should:

downshift intensity/complexity

reduce volume

or swap to a lower band (short-fast → medium → longer-contact)

3) Define “Units of Training” (what the app actually assembles)

Think LEGO pieces. Each unit has:

purpose (bandwidth / elastic output / exposure / robustness)

dose ranges by belt + phase + waveWeek

progressions/regressions

prerequisites

contraindications (esp. RTP stage constraints)

stop rules

Core unit types (MVP)

Strength+Plyo Unit A (your template)
Primer → Strength A + short contact → Strength B + longer contact → posterior chain → trunk

Strength+Plyo Unit B
Same skeleton but swapped emphasis (hinge focus vs squat focus, unilateral emphasis etc.)

Jump/Plyo/MBS Unit (field/gym)
Short block focused on movement skill + elastic qualities + throws (if relevant)

Running Unit

base / tempo / aerobic

speed exposure

COD/decel density exposures (when appropriate)

Important: treat “running unit” as part of exposure management, not conditioning punishment.

4) The data model the engine needs (simple, not bloated)
Athlete meta-data (minimum)

age

trainingAge (years of consistent structured training)

injury flags (esp recurrent hamstring/calf/groin; recent RTP)

current availability / training days

recent exposure counts (last 14d speed exposures, decel-heavy sessions)

movementQualityScore (coach/physio 1–5)

Tests (optional but ideal)

CMJ / RSImod

Drop jump RSI (if you have it)

hop LSI (triple hop etc.)

IMTP or strength proxy

5) How the engine decides what to output (selection logic)
Step-by-step selection

Classify belt (rule-based, coach override allowed)

Set weekly budgets (contacts, hard sets, speed touches) from belt + phase + waveWeek

Pick unit mix based on days available and sport context

Example standard: 2 Strength+Plyo units + 1 Running unit + optional Jump/MBS micro

Allocate budgets into units (per-session caps)

Apply constraints from stage overlay (RTP stage rules)

Generate plan JSON for UI rendering

Log outcomes (RPE, output drop %, pain response) → informs next week’s adjustment

B) Weekly budgets (belt + phase + waveWeek)
// src/engine/dosing.ts
import { Belt, Phase, WaveWeek, WeeklyBudgets } from "./types";

const baseBudgets: Record<Belt, WeeklyBudgets> = {
  WHITE: { plyoContacts: 110, hardLowerSets: 16, speedTouches: 1 },
  BLUE:  { plyoContacts: 140, hardLowerSets: 14, speedTouches: 1 },
  BLACK: { plyoContacts: 110, hardLowerSets: 10, speedTouches: 1 },
};

const phaseFactor: Record<Phase, { plyo: number; sets: number; speed: number }> = {
  PRESEASON_A:    { plyo: 1.05, sets: 1.10, speed: 1.0 },
  XMAS_BLOCK:     { plyo: 0.75, sets: 0.75, speed: 1.0 },
  PRESEASON_B:    { plyo: 1.00, sets: 0.95, speed: 1.2 },
  PRECOMP:        { plyo: 0.85, sets: 0.85, speed: 1.2 },
  INSEASON_EARLY: { plyo: 0.80, sets: 0.80, speed: 1.0 },
  INSEASON_MID:   { plyo: 0.80, sets: 0.80, speed: 1.0 },
  INSEASON_LATE:  { plyo: 0.70, sets: 0.70, speed: 1.0 },
  BYE_WEEK:       { plyo: 0.90, sets: 0.90, speed: 1.1 },
};

const waveFactor: Record<WaveWeek, number> = { 1: 1.0, 2: 0.95, 3: 0.75 };

export function getWeeklyBudgets(belt: Belt, phase: Phase, week: WaveWeek): WeeklyBudgets {
  const b = baseBudgets[belt];
  const p = phaseFactor[phase];
  const w = waveFactor[week];

  return {
    plyoContacts: Math.round(b.plyoContacts * p.plyo * w),
    hardLowerSets: Math.round(b.hardLowerSets * p.sets * w),
    speedTouches: Math.max(1, Math.round(b.speedTouches * p.speed)),
  };
}

C) Belt classification (rule-based + modifiers)
// src/engine/classification.ts
import { AthleteMeta, Belt } from "./types";

export function classifyBelt(meta: AthleteMeta): { belt: Belt; reasons: string[]; modifiers: string[] } {
  const reasons: string[] = [];
  const modifiers: string[] = [];

  let score = 0;

  // training age
  if (meta.trainingAgeYears >= 3) score += 2;
  else if (meta.trainingAgeYears >= 1) score += 1;
  else reasons.push("Low training age");

  // movement quality
  if (meta.movementQualityScore >= 4) score += 2;
  else if (meta.movementQualityScore === 3) score += 1;
  else reasons.push("Movement quality unstable at speed");

  // injury / RTP constraints
  if (meta.injuryFlags.recentRTP) {
    score -= 2;
    reasons.push("Recent RTP: cap reactive intensity/volume");
    modifiers.push("CAP_REACTIVE_CONTACTS");
  }
  if (meta.injuryFlags.recurrentHamstring || meta.injuryFlags.recurrentCalf || meta.injuryFlags.recurrentGroin) {
    score -= 1;
    modifiers.push("EXPOSURE_PROGRESSIONS_REQUIRED");
  }

  // exposure history
  if (meta.recentExposure14d.speedTouches === 0) modifiers.push("REINTRO_SPEED_PROGRESSIVELY");

  let belt: Belt = "WHITE";
  if (score >= 3) belt = "BLACK";
  else if (score >= 2) belt = "BLUE";

  return { belt, reasons, modifiers };
}

E) Planner: assemble a weekly plan from budgets + units
// src/engine/planner.ts
import { AthleteMeta, Phase, WaveWeek } from "./types";
import { classifyBelt } from "./classification";
import { getWeeklyBudgets } from "./dosing";
import { UNIT_LIBRARY } from "./unitLibrary";

export function buildWeeklyPlan(meta: AthleteMeta, phase: Phase, waveWeek: WaveWeek) {
  const { belt, reasons, modifiers } = classifyBelt(meta);
  const budgets = getWeeklyBudgets(belt, phase, waveWeek);

  // MVP selection: 2 strength+plyo + 1 running speed touch (if gym access)
  const sp = UNIT_LIBRARY.find(u => u.id === "SP_A")!;
  const run = UNIT_LIBRARY.find(u => u.id === "RUN_SPEED_TOUCH")!;

  const units = meta.availability.gymAccess
    ? [sp, sp, run]
    : [run]; // fallback if no gym (you can add bodyweight units later)

  return {
    belt,
    reasons,
    modifiers,
    phase,
    waveWeek,
    budgets,
    units,
    globalStopRules: [
      "If quality fails: reduce volume/density before reducing intensity.",
      "Capacity work is conditional, not blanket.",
    ],
  };
}
