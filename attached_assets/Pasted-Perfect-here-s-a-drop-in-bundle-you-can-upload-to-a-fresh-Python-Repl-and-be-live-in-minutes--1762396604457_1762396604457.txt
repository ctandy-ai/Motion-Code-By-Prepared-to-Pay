## If Replit AI is “ignoring” design requests or breaking functionality

Here’s how to stop it from mangling files and keep progress moving:

1. **Paste whole files, not snippets**
   The agent struggles with partial diffs. Replace the entire `pro.html` or `client.html` file in one go when you make changes.

2. **Pause AI for bulk edits**
   Toggle the AI helper off while you paste and run, then re-enable for smaller edits or Q&A.

3. **Lock formatting**
   Single-file HTML with inline Tailwind + JS is deliberate: fewer moving parts, fewer AI “fixes.” If you later split files, add a `.editorconfig` and stick to one formatter so AI is less tempted to reflow.

4. **Keep a /backup or GitHub mirror**
   Copy known-good versions into `/backup` or push to GitHub. If the AI breaks something, copy back instantly.

5. **Small commits; test after each change**
   Change one section → run → commit. The agent behaves better on tiny diffs.

6. **Don’t let AI touch the Run command**
   Keep:

   ```bash
   uvicorn app:app --host 0.0.0.0 --port 8000
   ```

   Pin this in the Run pane.
